# CS 300 Portfolio Submission

## About This Repository

This repository contains my portfolio artifacts for CS 300: Data Structures and Algorithms. The work included here shows my understanding of how different data structures can be used to solve a programming problem, as well as how algorithms can be designed to organize and display information efficiently.

## Portfolio Artifacts

For this course, I included the following artifacts:

* **Project One Runtime and Memory Analysis**
  This document compares different data structures, including vectors, hash tables, and binary search trees. It explains the runtime and memory use of each structure and discusses which one would be the best option for the course advising program.

* **Project Two Course Planner Code**
  This C++ program loads course information, stores the data, and prints the Computer Science courses in alphanumeric order. The program helps academic advisors access course information in an organized way.

## Reflection

The problem I was solving in these projects was how to design a program that could help ABC University advisors access course information for students. The program needed to load course data from a file, store that information, and allow the user to view courses in order or search for a specific course. The main goal was to choose the best data structure for organizing and retrieving the course information.

I approached the problem by comparing different data structures and thinking about how each one handled storing, searching, and printing data. Understanding data structures is important because the structure used can affect how fast and efficient a program is. For example, a vector is simple and easy to use, but searching through it can take longer if there is a lot of data. A hash table can make searching faster, while a binary search tree can keep data organized in sorted order. Looking at these options helped me understand why choosing the right data structure matters when designing software.

One roadblock I encountered was making sure the data from the file was loaded correctly and that the program could handle the course information properly. It was important to check that each line from the file was parsed correctly and that the course number, course title, and prerequisites were stored in the right places. I overcame this by breaking the project into smaller parts, testing each function, and reviewing the logic step by step.

This project expanded the way I think about designing software because it showed me that planning is just as important as writing code. Before this course, I mostly focused on getting a program to work. Now I think more about efficiency, organization, and how the program will handle larger amounts of data. I also learned that pseudocode and analysis help make the coding process easier because they give the program a clear direction before implementation begins.

My work on this project also helped me improve the way I write programs that are maintainable, readable, and adaptable. I focused on organizing the code into functions, using clear names, and keeping the logic easier to follow. This makes the program easier to update or fix later. If new features were added in the future, such as adding new courses or changing how courses are searched, the program would be easier to adapt because the code is organized and readable.

Overall, these projects helped me understand how data structures and algorithms are used in real software development. They also gave me more practice writing programs that are efficient, organized, and easier to maintain.
